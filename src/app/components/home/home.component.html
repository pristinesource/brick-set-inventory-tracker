⚠️ CRITICAL: Backup Your Data Regularly

Your inventory data is stored locally in your browser and could be lost permanently.

📥 Backup Instructions:

  1. Go to Settings → Data Management
  2. Click "Export Data" to download a backup file
  3. Save the file safely (cloud storage, email to yourself, etc.)
  4. Do this after every session!

🚨 Data Loss Scenarios:

  • Browser cache cleared: All data permanently lost
  • Browser reset/reinstall: All data permanently lost
  • Computer crash/replacement: All data permanently lost
  • Privacy mode/incognito: Data not saved between sessions
  • Browser storage limits exceeded: Data may be automatically deleted
  • System cleanup tools: May delete browser data

📝 Remember: This app runs entirely in your browser. No data is stored on our servers. Regular backups are your only protection against data loss!

📚 How to Use This App

🚀 Getting Started

  1. Browse available sets in the Sets section
  2. Click "Add to My Sets" for sets you own or want to track
  3. Select the correct version if multiple versions exist
  4. Click "View Inventory" to start tracking your parts
  5. Mark parts as owned by updating quantities
  6. Use the Missing Parts view to see what you still need

✨ Key Features

  • Part Tracking: Mark individual parts as owned
  • Minifigure Tracking: Track minifigures and their parts separately
  • Spare Parts: Optional inclusion in progress calculations
  • Progress Visualization: See completion percentages
  • Missing Parts View: Consolidated view of all missing items
  • Undo Functionality: Undo recent changes (Ctrl/Cmd+Z)
  • Sorting & Filtering: Organize parts by various criteria

💡 Tips & Best Practices

📋 Organization

  • Update quantities as you sort physical parts
  • Use the search function to find specific parts quickly
  • Sort by completion to focus on missing parts first
  • Check minifigure parts separately - they can be detailed

💾 Data Safety

  • Export data after every session
  • Store backups in multiple safe locations
  • Test imports periodically to ensure backups work
  • Don't rely solely on browser storage

⚡ Performance

  • Large sets may take time to load initially
  • Data is cached for faster subsequent loads
  • Use list view for large sets to improve scrolling
  • Adjust image size in settings for better performance
